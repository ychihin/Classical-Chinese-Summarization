# Classical-Chinese-Summarization

This repository contains experiments for classical-to-modern Chinese processing, with a current emphasis on:

- round-trip translation from modern Chinese to English and back to modern Chinese
- automatic evaluation of round-trip fidelity
- baseline summarization notebooks and utilities

Reference paper:
- [Classical Chinese Summarization](https://arxiv.org/abs/2308.00240)

## Project Layout

```text
│   └── processed/

│       ├── translated_back/    # translated files translated back from en -> zh-cn

```

## Data Convention

Each text unit is stored in a directory that typically contains:

- `source.txt`: classical Chinese source text
- `target.txt`: modern Chinese reference text
- `bitext.txt`: paired display of source and target

For round-trip translation:

1. `data/raw/<corpus>/.../target.txt` is treated as the original modern Chinese reference.
2. `data/processed/translated/<corpus>/.../target.txt` stores the English translation.
3. `data/processed/translated_back/<corpus>/.../target.txt` stores the back-translated modern Chinese output.

The evaluation scripts compare:

- original modern Chinese: `data/raw/.../target.txt`
- round-trip output: `data/processed/translated_back/.../target.txt`

## Translation Workflow

Core translation utilities live in [src/translation.py](/Users/aojiang/nltk_data/taiyichen/Classical-Chinese-Summarization/src/translation.py) and use `googletrans`.

Main functions:

- `translate(source_path, src_lang, dest_lang)`
  - translates one `target.txt`
  - returns elapsed time, line count, and translated lines
- `one_way_translate(input_root, output_root, source_lang, target_lang)`
  - recursively scans `target.txt` files
  - skips files that already exist in the output directory
  - preserves relative paths under the destination root

Example round-trip entrypoint:

- [src/round_trip.py](/Users/aojiang/nltk_data/taiyichen/Classical-Chinese-Summarization/src/round_trip.py)

It currently demonstrates:

- `zh-cn -> en` into `data/processed/translated/<corpus>`
- `en -> zh-cn` into `data/processed/translated_back/<corpus>`

## Evaluation

### 1. Existing ROUGE Metrics

[models/evaluation/rouge.py](/Users/aojiang/nltk_data/taiyichen/Classical-Chinese-Summarization/models/evaluation/rouge.py) contains:

- `rouge_1`
- `rouge_2`
- `rouge_l`

### 2. Additional Fidelity Metrics

[models/evaluation/metrics.py](/Users/aojiang/nltk_data/taiyichen/Classical-Chinese-Summarization/models/evaluation/metrics.py) adds:

- `chrF`
- `bleu`
- `edit_similarity`
- `length_ratio`
- `embedding_similarity` (optional, requires extra dependency)
- `score_all`

These metrics are intended to give a broader view of round-trip quality than ROUGE alone.

### 3. Single-Corpus Evaluation

Use [scripts/evaluate_round_trip.py](/Users/aojiang/nltk_data/taiyichen/Classical-Chinese-Summarization/scripts/evaluate_round_trip.py) to evaluate one corpus.

Default configuration evaluates:

- source: `data/raw/晋书`
- back-translation: `data/processed/translated_back/晋书`
- output: `data/processed/evaluation/晋书_round_trip`

Run:

```bash
python3 scripts/evaluate_round_trip.py
```

Optional arguments:

```bash
python3 scripts/evaluate_round_trip.py \
  --source-root data/raw/晋书 \
  --back-root data/processed/translated_back/晋书 \
  --output-root data/processed/evaluation/晋书_round_trip
```

Outputs:

- `line_metrics.csv`
- `file_summary.csv`
- `summary.json`
- `summary.md`

### 4. Multi-Corpus Evaluation

Use [scripts/evaluate_all_round_trip.py](/Users/aojiang/nltk_data/taiyichen/Classical-Chinese-Summarization/scripts/evaluate_all_round_trip.py) to evaluate every corpus that has:

- raw modern Chinese references under `data/raw`
- back-translated files under `data/processed/translated_back`

Run:

```bash
python3 scripts/evaluate_all_round_trip.py
```

Default output directory:

- `data/processed/evaluation/all_round_trip`

Generated files:

- `corpus_inventory.csv`
- `corpus_metrics.csv`
- `line_metrics.csv`
- `summary.json`
- `summary.md`

## Environment Notes

The repository currently does not maintain a complete `requirements.txt`. Based on the current code, you will likely need at least:

```bash
pip install googletrans==4.0.0rc1
```

If you want to use optional embedding similarity from `metrics.py`, you will also need:

```bash
pip install sentence-transformers
```

## Typical Workflow

1. Add or sync raw bilingual data into `data/raw/<corpus>`.
2. Run translation into `data/processed/translated/<corpus>`.
3. Run back-translation into `data/processed/translated_back/<corpus>`.
4. Run evaluation scripts to generate CSV/JSON/Markdown summaries.
5. Inspect `summary.md` for quick reporting and `corpus_metrics.csv` / `file_summary.csv` for deeper analysis.

## Notes for Contributors

- The directory structure matters: scripts rely on matching relative paths ending in `target.txt`.
- Evaluation is reference-based and assumes `data/raw/.../target.txt` is the modern Chinese gold text.
- Missing back-translated files are surfaced in the generated summaries and affect coverage metrics.
